## Особенности реализации

### Архитектура ACC
Процессор обладает одним основным регистром — **Аккумулятором**. 

### Память (Harvard)
*   **Instruction Memory**: Память с командами.
*   **Data Memory**: Память данных. Содержит переменные, константы и строки.

### Система прерываний
Механизм прерываний для обработки асинхронных событий.
1. При возникновении прерывания текущее состояние (IP, AC, PS) сохраняется.
2. Процессор переходит по адресу из таблицы векторов прерываний.
3. Команда возврата из прерывания восстанавливает контекст.

## Язык программирования

### Синтаксис
Язык является диалектом Lisp, построенным на основе S-выражений. Программа представляет собой последовательность выражений, которые могут быть либо атомарными значениями, либо списками. В списках используется префиксная нотация: первый элемент является оператором или функцией, остальные — аргументами.

**Форма Бэкуса-Наура:**

```bnf
<program>          ::= <expression> | <expression> <program>

<expression>       ::= <atom> | <list>

<list>             ::= "(" <list_body> ")"

<list_body>        ::= <special_form> 
                     | <function_call> 
                     | <operator_call>

<special_form>     ::= <def_form>
                     | <set_form>
                     | <if_form>
                     | <while_form>
                     | <block_form>
                     | <lambda_form>
                     | <io_form>
                     | <trap_form>

<def_form>         ::= "def" <symbol> <expression>
<set_form>         ::= "set" <symbol> <expression>
<if_form>          ::= "if" <expression> <expression> <expression>
<while_form>       ::= "while" <expression> <expression>
<block_form>       ::= "block" <expression_list>
<lambda_form>      ::= "lambda" "(" <parameter_list> ")" <expression>
<io_form>          ::= "out" <number> <expression> | "in" <number>
<trap_form>        ::= "trap" <number>

<function_call>    ::= <symbol> <expression_list>
<operator_call>    ::= <math_operator> <expression_list>

<expression_list>  ::= <expression> <expression_list> | ""
<parameter_list>   ::= <symbol> <parameter_list> | ""

<math_operator>    ::= "+" | "-" | "*" | "/" | "%" | "^" | "<" | ">" | "="

<atom>             ::= <number> | <symbol> | <string> | <boolean>

<symbol>           ::= [a-zA-Z_][a-zA-Z0-9_!?-]*
<number>           ::= "-"? [0-9]+
<string>           ::= '"' [^"]* '"'
<boolean>          ::= "#t" | "#f"
```

### Семантика

#### Стратегия вычислений
Используется **аппликативный порядок вычислений**. Все аргументы выражения вычисляются рекурсивно до того, как к ним будет применен оператор или вызвана функция.
#### Области видимости
*   **Глобальная область видимости**: Переменные, объявленные через `def` вне тела функций, доступны во всей программе.
*   **Локальная область видимости**: Параметры, объявленные в `lambda`, доступны только внутри тела данной функции. 
*   **Затенение (Shadowing)**: Локальные переменные перекрывают одноименные глобальные переменные на время выполнения функции.

#### Типизация и литералы
Язык обладает динамической типизацией. Объекты данных в памяти имеют фиксированный размер машинного слова (32 бита).
*   **Number**: 32-битные целые знаковые числа.
*   **Boolean**: Представлен литералами `#t` и`#f`.
*   **String/cstr)**: Null-terminated последовательности байт.
*   **Symbol**: Имена переменных и функций.

#### Особенности управляющих конструкций
*   `def`: Резервирует место в памяти данных и связывает его с именем.
*   `set`: Изменяет значение по адресу существующей переменной.
*   `block`: Группирует выражения. Выполняет их последовательно, результатом блока является результат последнего выражения.
*   `lambda`: Создает анонимную функцию. При вызове функции состояние процессора (адрес возврата) сохраняется для обеспечения возможности **рекурсии**.
*   `trap`: Генерирует программное прерывание.
### Примеры кода

**1. Работа с условиями и блоками:**
```lisp
(def x -1)
(if #t
    (block
        (out 0 "True block")
        (set x 1))
    (block
        (out 0 "False block")
        (set x 0)))
(out 0 x)
```

**2. Цикл и арифметика:**
```lisp
(def i 0)
(while (< i 10)
    (block
        (set i (+ i 1))
        (out 0 i)))
```

**3. Определение функции и её вызов:**
```lisp
(def summ (lambda (x y) (block (out 0 (+ x y)))))
(summ 5 10) ; => 15
```

## Система команд

Процессор использует 32-битные инструкции с фиксированным форматом:
- **Opcode (8 бит)**: Код операции.
- **Operand (24 бита)**: Адрес памяти (Addr) или непосредственное значение (Immediate).

### Список команд:

| Команда | Код  | Описание                                          |
| :---    | :--- |:--------------------------------------------------|
| **LD**  | 0x01 | AC <- Mem[addr]                                   |
| **ST**  | 0x02 | Mem[addr] <- AC                                   |
| **LDI** | 0x03 | AC <- imm (Прямая загрузкка операнда)             |
| **ADD** | 0x10 | AC <- AC + Mem[addr]                              |
| **SUB** | 0x11 | AC <- AC - Mem[addr]                              |
| **MUL** | 0x12 | AC <- AC * Mem[addr]                              |
| **MOD** | 0x14 | AC <- AC % Mem[addr]                              |
| **CMP** | 0x20 | Выставить флаги результата AC - Mem[addr]         |
| **JMP** | 0x30 | PC <- addr                                        |
| **JZ**  | 0x31 | PC <- addr (если флаг Z=1)                        |
| **JNZ** | 0x32 | PC <- addr (если флаг Z=0)                        |
| **CALL**| 0x40 | Push(PC); PC <- addr                              |
| **RET** | 0x41 | PC <- Pop()                                       |
| **PUSH**| 0x42 | Push(AC)                                          |
| **POP** | 0x43 | AC <- Pop()                                       |
| **IN**  | 0x50 | AC <- Port[imm]                                   |
| **OUT** | 0x51 | Port[imm] <- AC                                   |
| **TRAP**| 0x60 | Вызов системного прерывания                       |
| **IRET**| 0x61 | Возврат из прерывания (восстановление PC, AC, PS) |
| **HLT** | 0xFF | Остановка процессора                              |

### Формат бинарного представления
Инструкция упаковывается в 4 байта (Big Endian):
`[Opcode (8 bits)] [Argument (24 bits)]`
